Demo 4

Create, Read, Update, Delete

What We Will Learn

  • How to add a row to a table
  • How to update a row
  • How to delete a row

The most important demo for real data collection!

What Does the Plugin Do?

  • Get plot ID from parent plugin
  • Query the (non-geographic) table plot_header with plot_id
  • If no row found: Create new template row
  • Fill form with row values
  • Track changes → Save button shows state
  • Save button → Commit all values to plot_header

⚠️ Warning

NEVER bypass the QField API!

NEVER access through the data provider!

Why?

  • QField Sync is based on QGIS Offline Editing
  • Direct changes are not detected
  • Synchronization will overwrite your data

Always use FeatureUtils & LayerUtils!

CREATE: Insert Feature

// 1. Start editing layer.startEditing() // 2. Create empty feature var newFeature = FeatureUtils.createFeature(layer) // 3. Set attributes newFeature.setAttribute("f_uid", StringUtils.createUuid()) newFeature.setAttribute("plot_id", plotId) newFeature.setAttribute("log_date", new Date().toISOString()) // 4. Add feature to layer LayerUtils.addFeature(layer, newFeature) // 5. Commit changes layer.commitChanges() return newFeature

READ: Get Feature

var expression = "plot_id = '" + plotId + "'" var iterator = LayerUtils.createFeatureIteratorFromExpression(layer, expression) if (iterator.hasNext()) { currentFeature = iterator.next() iterator.close() // CRITICAL! return currentFeature } iterator.close() // Always close!

ALWAYS close iterator!

UPDATE: Update Feature

// 1. Start editing layer.startEditing() // 2. Get feature ID and fields var fid = currentFeature.id var fields = currentFeature.fields // 3. For each field: get index and change value for (var fieldName in fieldValueMap) { var value = fieldValueMap[fieldName] var fieldIndex = fields.indexOf(fieldName) if (fieldIndex >= 0) { layer.changeAttributeValue(fid, fieldIndex, value) } } // 4. Commit changes layer.commitChanges()

DELETE: Delete Feature

// 1. Start editing // 1. Start editing layer.startEditing() // 2. Delete feature var success = layer.deleteFeature(featureId) if (!success) { // 3a. On error: Rollback layer.rollBack() return false } // 3b. On success: Commit var commitSuccess = layer.commitChanges() if (!commitSuccess) { layer.rollBack() return false } return true

Important Warning: Field Index

NEVER use hard-coded indices!

Index in QGIS is not the same as index in QField.

❌ WRONG:

value = feature.attribute(3)

✅ CORRECT:

value = feature.attribute("plot_id") // Use field name! // Or for updates: var fieldIndex = fields.indexOf("plot_id") layer.changeAttributeValue(fid, fieldIndex, value)

Model View Controller Architecture

  • Model: d4_form_data_model.qml

    Configuration of form structure: labels, field names (id), widget settings

  • View: d4_headerpage.qml, d4_form_section.qml, d4_form_field.qml

    Register with the controller

  • Controller: d4_form_controller.qml

    Coordinates between view and model

  • Controller: d4_data_controller.qml

    Handles QField API interactions

FormController Pattern

  • 1. Widget elements register with the controller
  • 2. FormController collects values from all widgets
  • 3. FormController passes values to DataController
  • 4. DataController executes QField API operations

Repeater for Dynamic Forms

Repeater { model: FormDataModel.groupBoxes delegate: FormSection { width: parent.width sectionData: modelData controller: headerPage.formController } }
  • Configuration defined in DataModel
  • Repeater iterates and creates widget elements
  • Widget elements register on creation
  • No for-loops in QML!

Property Propagation through Loader

// Define in parent Loader Loader { id: tabWidgetLoader property string plotId: pluginFrame.plotId source: "d4_tabwidget.qml" } // Use in loaded component // d4_tabwidget.qml Item { property string plotId: parent.plotId // From Loader! }
